# Клеточный автомат с настраиваемыми правилами и выбором цветов

Проект представляет собой интерактивную симуляцию многодветного клеточного автомата, вдохновлённую генерацией биомов. Пользователь может в реальном времени настраивать правила перехода между двумя выбранными цветами, изменять их вероятности, а также менять отображаемую цветовую пару.

![Скриншот](https://screenshot.png)

## Возможности

- Два активных цвета – пользователь выбирает пару цветов (A и B), которые взаимодействуют между собой. Все остальные цвета игнорируются в симуляции, но могут присутствовать на поле (при случайном заполнении или рисовании).
- Настройка правил перехода – для каждого направления (A → B и B → A) задаются:
  - список количеств соседей того же цвета, при котором срабатывает правило;
  - вероятность перехода в другой цвет (от 0 до 1).
- Графический интерфейс настройки – модальное окно с ползунками и текстовыми полями (библиотека raygui).
- Управление симуляцией: Play/Pause (пробел или кнопка), Step (один шаг, клавиша N), Clear (клавиша C), Random (клавиша R).
- Редактирование поля мышью: левая кнопка – цвет A, правая – цвет B.
- Навигация: колесо мыши – масштабирование, стрелки – панорамирование.
- Выбор отображаемых цветов (кнопки A: и B:) – меняются только отображаемые цвета, правила перестраиваются для новой пары.

## Технологии

- Язык: C++17
- Графика: [Raylib 6.0](https://www.raylib.com/)
- GUI: raygui (header‑only)
- Компилятор: MinGW-w64 (ucrt64), GCC или Clang

## Сборка и запуск

### Требования

- Компилятор с поддержкой C++17
- Библиотека Raylib (скомпилированная для вашей системы)
- **Для Windows (MSYS2)** рекомендуется установить MSYS2 и пакеты `mingw-w64-ucrt-x86_64-gcc` и `mingw-w64-ucrt-x86_64-raylib`

### 1. Компиляция через Makefile (рекомендуемый способ)

1. Скачайте и распакуйте Raylib, например, в `C:/raylib/raylib-6.0_win64_mingw-w64`.
2. Отредактируйте `Makefile`, указав правильные пути к заголовочным файлам и библиотеке Raylib.  
   Пример для Windows (MSYS2):
   ```makefile
   RAYLIB_PATH = C:/raylib/raylib-6.0_win64_mingw-w64
   CXXFLAGS = -I"$(RAYLIB_PATH)/include"
   LDFLAGS = -L"$(RAYLIB_PATH)/lib" -lraylib -lopengl32 -lgdi32 -lwinmm
В терминале (MSYS2 UCRT64) перейдите в папку проекта и выполните:

bash
make
Убедитесь, что raylib.dll находится в той же папке, что и game_of_life.exe (или в PATH).

Запустите:

bash
./game_of_life.exe
2. Ручная компиляция (если make не работает)
Используйте следующую команду (пути должны указывать на вашу установку Raylib):

bash
g++ main.cpp Cell/Cell.cpp Rules/Rules.cpp GameGrid/GameGrid.cpp GameRenderer/GameRenderer.cpp Game/Game.cpp raygui_impl.cpp -o game_of_life.exe -I"C:/raylib/raylib-6.0_win64_mingw-w64/include" -L"C:/raylib/raylib-6.0_win64_mingw-w64/lib" -lraylib -lopengl32 -lgdi32 -lwinmm
Примечание: Если вы используете другую операционную систему (Linux, macOS), замените флаги -lopengl32 -lgdi32 -lwinmm на соответствующие (обычно -lraylib -lm).

3. Примечание о raygui.h
Файл raygui.h уже включён в папку Game/. Реализация подключается через raygui_impl.cpp, который определяет RAYGUI_IMPLEMENTATION и включает raygui.h. Ничего дополнительно настраивать не нужно.

Управление
Действие	Клавиша / Мышь
Play / Pause	Пробел (или кнопка Play/Pause)
Шаг симуляции	N (или кнопка Step)
Очистить поле	C (или кнопка Clear)
Случайное заполнение	R (или кнопка Random)
Открыть окно правил	S (или кнопка Rules)
Увеличить / уменьшить	Колёсико мыши
Панорамирование	Стрелки
Поставить цвет A	Левая кнопка мыши
Поставить цвет B	Правая кнопка мыши
Сменить цвет A	Кнопка "A:"
Сменить цвет B	Кнопка "B:"
Выход	ESC
Настройка правил
При нажатии кнопки Rules (или клавиши S) открывается модальное окно. В нём можно:

Указать список количеств соседей (через пробел) для каждого направления.

Изменить вероятность перехода (ползунок).

Сохранить изменения или отменить.

После сохранения правила применяются к текущей паре активных цветов (A и B). При смене A или B правила автоматически перестраиваются для новой пары (с сохранёнными параметрами).
